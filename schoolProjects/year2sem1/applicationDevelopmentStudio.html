How it works.

Part 1.

Using JOptionPane, a window will appear displaying prompts and messages to the user. The user will be presented with a prompt to enter their account number. There are three accounts numbers that I set, "123", "321" and "456". In this case, I entered 123. If I entered anything else besider 123, 321 and 456, a error message will pop up telling me that there is no such account then it will prompt me to enter again.

When a valid account number is entered, the next thing the machine will do is prompt for the password. If the password is incorrect, a message will tell the user that the password do not match and prompt the user to try again.

After the correct password has been entered, a whole list of things that they can do is displayed. Users can enter a number based on the list given. If the user enter any number that is not in the list, they will be given an error message telling them to only enter numbers in the range of 1 to 7.

For the first option (check the balance), user will be displayed their balance. Pressing the ok button will bring the user back to the list of option window.

The second option (withdraw), prompts the user to enter the amount that they would like to withdraw from their account. In this example, I withdraw $500 dollars. It will then show the remainding balance in the account after withdrawer.

In the case of the user entering an amount that they want to withdraw from their account more than what they have in their account, an error meesage will be shown.

The third option (deposit), prompts the user to enter the amount that they would like to deposit from their account. In this example, I deposited $1000. The remaining balance will be displayed afterwards.

The forth option (transfer funds), prompts the user to enter the account number they want the money to be transffered to then the amount to be transffered. In this example, I transffered $200 to the account 456. The remaining balance will be displayed afterwards.

When the transfer of funds is sucessful, the machine will show the remainding balance for the account. In the case of, entering a account that does not exist an error message will be shown.

For the fifth option (switch accounts), they will be prompted to enter the account number that they want to switch to and the password for that account.

In this example, I switched to the 456 account which I previously transffered $200 to from the 123 account. The orginal amount of money account 456 have is $3000 but after using option one to check, the balance indeed has increased by $200.

The sixth option (change password), the user is first prompted to enter their current password then the new password that they want and lastly, entering the new password once again to confirm.

In the situation of the user entering the wrong password, an error message will be shown. When the password is successfully change, a message will be displayed to inform the user.

When the user feel like they are done with whatever they wanted to do, they can use the last option (exit) which they will be presented with a thank you message.

Part 2.

In this part of the project, I made a Swing GUI using GUI builder in Netbeans. The ATM machine has the same features as previously but there are some additional features that I will be pointing out below.

When the user first run the program, they can use the number pad to key in their account number and pin number. All buttons besides the number pad button and the exit button is disabled and only would be enabled after successful entering of account and pin number. The withdraw, deposit and transfer funds is also not shown at the side.

I have added a feature where there is two types of accounts, the savings and fixed deposit account. Both types of account has interest that the user can benefit from in this bank. However, the fixed deposit has a higher rate of interest. The user wont be able to withdraw, deposit and transfer for a fixed deposit account.

Another feature I have added is the creating, deleting, reading and writing of text files. With this, I am able to save the user's transactions so that when they log out and log back in again the balance in the accounts will continue from their previous use. I also am able to create a text file to store the history of the user's transactions which I will be showing below.

In this example, I have done a withdrawal, deposit and transfer using 1234567890 account. If the file transactionHistory1234567890.txt does not exist in the project folder, the program would create it a record down the transactions I've made. Because I have transffered money to 0987654321 account using 1234567890 account, transactionHistory0987654321.txt will be created also.
