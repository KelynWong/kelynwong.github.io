Website & mobile application

An application to find the best advertising option

In this project, we combined our previous module's knowledge to make a website & mobile application for the ficticious company called JiBaBoom. They requested that the application is able to retrieve data from different company's advertising options for viewing. Then they are also able to pick out several advertising options and key into the system to calculate which of those options chosen is the best. The best option is determined by the most amount of audience reached with the lowest price.

How it works.

This is the basic data viewer, where the user is able to view the options from all companies. They can chose to input values in the fields to filter either by company id or audience reach or both. The data is taken from a database in elephantsql.

This is the basic result viewer, where the user is able to enter the option id they have picked out from the data viewer to compare which is the best option. They are also required to enter their budget.

This is the advanced data viewer, the functionality of this page is the same as the basic one but the difference is that there is an extra filter of cost and the data that is retrieved from the database is for the advanced options. I'll be talking about the difference between the basic and advance options later on.

This is the advanced result viewer, the functionality of this page is the same as the basic one. Now, there is a difference between the basic and advanced options. The basic options allows the option to be paid in partial amount while the advanced option does not allow that. For example, I have four options with the audience reach of 1,2,3,4 and cost of 4,3,2,1 accordingly. I then keyed in a budget of 5. In the basic case, the result will be that I can get 8.5 audience with 5 cost because I can take the 4th, 3th and a partial of the 2nd option. In the advance case, the result will be that I can get 7 audience with 3 cost because I can only take the 4th and 3rd option.
