How it works.

Part 1.

This part of the project is individual work. I have to design a database based on the requirements on product display on the website. This online grocery shop would be similaar to the NTUC FairPrice. These are the requirments:

1) As an online user, I must be able to browse the products based on product category. For each product, the product name, a short product description, price and promotion/discount information must be displayed. For some products, the weight/volume/size information should also be displayed.

2) As the store owner, I must be able to update the regular price of each product at any time.

3) As the store owner, I must able to update a product such that product is no longer available on the online store. (Product record should not be deleted)

4) As the store owner, I must be able to update the discount information and promotion information at any time.

With the requirments in mind, this is the the ERD of the database that I have designed. To view the detailed explanation click here.

Besides designing the database, I came up with four select statments to prove that my database is able to meet the requirements requested so that the data can be shown on the website. The first statement selects all products that are on offer, displaying the product name and the price.

This is the second statement that selects all products and arranging them in accending order of the price.

This is the third statement that selects the count of products belonging to the household/detergent category and has the brand of either breeze or dynamo.

Lastly, this statement selects the current price of a specific product.

Part 2.

For this part of the project, we worked in a team of four to design a data warehouse. Firstly, we are given a ERD digram and we had to create the sql script on the OLTP database table creation.

We analysed the OLTP to figure out which schema for the OLAP to use. After careful considerations, we decided to use a star schema instead of the snowflake one.

We then proceed to create the data warehouse according to the diagram. Next, we were give data from various to be put into the data warehouse for storage. One from the OLTP database itself, one from a json file and another from a csv file. However, before intregrating all those data into the data warehouse, we checked through the data to find any data inconsistencies. One example of the inconsistencies that we found were that some country name were spelled in full so we corrected that before it goes into the data warehouse.

After sufficent data is imported into the OLAP, we had to come up with insightful queries retreiving data that would be helpful in improving the business of the company. Our first sql statement relates to sales where we list the least profitable product and order by the office that contributed to the most loss.

This second statement relates to staff/offices where we select the best performing offiec based on total sale during a certain period of time.

The third statement relates to seasons of sales where we select the percentage of the order statuses over total order over the quarters of each year.

The last statement relates to orders/customers where we compute the the money spent by each customer based on their orders. We also show each customer's spending as a percentage of total spending by all customers.

Moving on, there is a individual component to this part of the project. Extending from the data warehouse, we had to transfer the data to databricks, a data analytics platform. Firstly, I exported each table's data in the data warehouse as a csv file.

After having created an account in databricks, I imported the csv files into the filestore.

I then proceeded to create a workspace in databricks and created the database and tables using the cmd. I excluded the timeDIM table in databricks as it was not necessary.

When all tables are created and the data is correctly imported in the tables accordingly, I came up with one more insightful query for the business. This query enablea the owner to see an overview of all the offices and employees and is able to pinpoint where and whom their sales revenue is coming from.
